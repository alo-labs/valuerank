Methodology
Primary-source update logic
This version uses current public benchmark pages only. No historical cost panels, no old SWE-bench Verified split, and no mixed-vintage benchmark pools.

1. Cohort definition

The ranked set is the current public Artificial Analysis coding-agent cohort: 21 agent variants spanning Claude Code, Codex, Cursor CLI, Gemini CLI, and Opencode.

2. Quality signals

Artificial Analysis Coding Agent Index is the main quality anchor. It already averages three benchmarks: SWE-Bench-Pro-Hard-AA, Terminal-Bench v2, and SWE-Atlas-QnA.

3. Gap treatment

Official Terminal Bench 2.1 currently covers only 8 of the 21 ranked rows after direct and family-level agent/model matching. Missing TB2.1 coverage is neutralized at 50, not imputed upward or downward.

Scoring Formula

Default weighting keeps Artificial Analysis clearly primary and limits TB2.1 double-counting, since Terminal-Bench v2 is already inside the AA index.

Overall = costWeight * CostNorm + (1 - costWeight) * (0.80 * AAIndexNorm + 0.20 * TB2.1Norm) Default costWeight = 0.25 Therefore: Overall = 0.25 * CostNorm + 0.60 * AAIndexNorm + 0.15 * TB2.1Norm

Family-level TB2.1 aliases

Terminal Bench 2.1 publishes agent-plus-model families, not Artificial Analysis effort settings. The same official TB2.1 score is therefore reused for matching effort variants within the same family.

  • `Codex CLI + GPT-5.5` -> `Codex + GPT-5.5 (medium)` and `Codex + GPT-5.5 (xhigh)`
  • `Claude Code + Claude Opus 4.8` -> `Claude Code + Opus 4.8 (medium)` and `(max)`
  • `Claude Code + Claude Opus 4.7` -> `Claude Code + Opus 4.7 (medium)` and `(max)`
  • `Claude Code + GLM 5.1` -> `Claude Code + GLM-5.1`

Why cost stays in the formula

Artificial Analysis reports mean API cost per task using current per-token pricing, including cache behavior where exposed. That is the cleanest uniform public cost basis across the entire cohort. If cost is irrelevant for your use case, drag the slider to `0%` for a pure quality ordering.

Data Gaps
What is fully covered and what is still missing
The benchmark update is current, but the public ecosystem is still incomplete. ValueRank should show that explicitly instead of pretending the gaps do not exist.

Covered now

  • Artificial Analysis Coding Agent Index for all 21 ranked variants.
  • Artificial Analysis per-benchmark component scores: Terminal-Bench v2, SWE-Bench-Pro-Hard-AA, and SWE-Atlas-QnA.
  • Artificial Analysis mean API cost, wall time, and total token telemetry.
  • Official Terminal Bench 2.1 verified entries for 11 public submissions, with 8 usable matches into the AA cohort.

Still missing publicly

  • No official Terminal Bench 2.1 entry yet for most Cursor CLI, Opencode, Kimi, Qwen, DeepSeek, or Sonnet 4.6 AA variants.
  • No current public benchmark page that cleanly maps all AA effort levels to their own distinct TB2.1 scores.
  • No need to reuse the retired TB2.0 plus SWE-bench Verified split; that would create mixed-vintage comparisons.

Source timestamps

Artificial Analysis page rechecked June 11, 2026. Terminal Bench 2.1 page rechecked June 11, 2026. Rankings on this page reflect those live public pages on that date.

Interpretation rule

If two agents are close on overall score, inspect the underlying benchmark mix. Repository reasoning, code patching, and terminal workflow are not the same skill, and Artificial Analysis exposes that breakdown directly.
