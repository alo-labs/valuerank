Strategic Insights
Four empirical findings that determine how to interpret and use these rankings.
Scaffold Engineering Premium
~10 pp
Same model, different scaffold = 10 percentage-point gap at the frontier. ForgeCode + Gemini 3.1 Pro scores 78.4% on TB2.0 while Google's own direct Gemini submission scores ~68.5% — same weights, different scaffold. Choosing the best model is not enough.
Open-Source Near-Parity on SWE-V
3.6 pp
OpenHands v3 achieves 68.4% vs. Augment Code's 72.0% on the same Claude Opus 4.6 base model. The 3.6 pp gap is the cost of Augment's proprietary BM25 + code-graph retrieval scaffold. For teams preferring BYOK economics, OpenHands is the rational choice.
Long-Horizon Capability Cliff
74% → 11%
FeatureBench (ICLR 2026): agents scoring 68–74% on SWE-V collapse to 5–13% on multi-file feature development tasks. The cliff is consistent across all tested agents. Current benchmark scores predict contained task performance only — not "implement this 3-week feature across 15 files."
Devin 2.0 Brand vs. Benchmark Gap
Rank 8
Despite Cognition AI's high profile and reported valuation, Devin 2.0 sits at SWE-V rank 8 at 45.8% — 22.6 pp below OpenHands v3 (68.4%) and below three open-source systems. Differentiation is product experience and team workflow tooling, not benchmark performance.

FAQ
Cost Methodology & Data Provenance
How the run-cost figures are sourced, how reliable they are, and what to do when the data is missing or proxied.
The Cost column shows each entry's underlying model's Artificial Analysis Eval Cost in USD — the dollar cost to run AA's standardized intelligence-evaluation suite on that model (HLE, GPQA Diamond, AIME, MMLU-Pro, BrowseComp, AA-Omniscience, etc.). It is NOT the cost to run TB2.0 or SWE-bench Verified. Methodology choice: AA Eval Cost is the only cost figure with uniform coverage across our model pool and identical methodology across rows. Using it as the Cost column makes value comparisons internally consistent — when you see Opus 4.6 ($4,970) priced at 5.5× Gemini 3.1 Pro ($892), that ratio holds across all rows and means the same thing in every row. Absolute dollars are eval-suite-relative, not benchmark-run-realistic — for actual measured run cost, see the Measured Cost research panels below each table.
One auxiliary table presents actual per-trial token usage for the 7 TB2.0 entries where we recovered measured data. Source: huggingface.co/datasets/harborframework/terminal-bench-2-leaderboard — the official tbench.ai submission archive. We git-cloned the 8.1 GB dataset, parsed all 16,800+ per-trial result.json files, and aggregated n_input_tokens / n_output_tokens / n_cache_tokens / cost_usd across the 13 of 37 submissions that populated those fields. For entries whose own scaffold left tokens null, we transfer the per-model median from same-model sibling submissions.

No SWE-V research panel: an earlier version of this page included one (sourced from swe-rebench.com), but the SWE-V table itself was rebuilt April 28, 2026 to reflect the actual swebench.com primary leaderboard — and the new top-15 entries are dominated by proprietary scaffolds (TRAE, Atlassian Rovo Dev, Warp, Harness AI, ACoder) whose underlying models aren't disclosed. Per-token research data isn't publicly available for those entries. swe-rebench.com remains a separate, related leaderboard accessible from the LLM page (/#swe-rebench) — but it covers a different 57-task rolling window, not SWE-V's 500-task fixed set, so we don't conflate them here.

The TB2.0 panel is auxiliary, not the primary ranking: cross-scaffold variance is 3.5× even within the same model (e.g. Terminus-KIRA + Opus 4.6 = $1.02/trial vs. Mux + Opus 4.6 = $0.29/trial), so per-trial figures are a model-level baseline, not a scaffold-specific prediction.
Three reasons we kept measured cost as auxiliary research rather than primary ranking. (1) Coverage is partial. Only 7 of 11 TB2.0 entries have measurable data; the rest would need an AA-proxy fallback, mixing two methodologies in the same column — which is misleading because the absolute scale gap between AA-proxy and measured values (5–20×) is benchmark-architectural, not efficiency-driven. (For SWE-V, the Apr 28 data correction replaced the 13 third-party entries with 15 actual swebench.com top entries, most of which are proprietary scaffolds without measurable token data — making a SWE-V research panel infeasible.) (2) Scaffold variance is hidden. Transferring a per-model median to multiple scaffolds (because their own submissions left tokens null) papers over the 3.5× spread we observed. (3) Disclosed-vs-computed gap unexplained. Anthropic submissions report $0.530/trial but rate-card × tokens math gives $2.39/trial — a 4× discrepancy attributable to caching architecture or batch discounts we can't model. The auxiliary panel preserves the research integrity (here's what we measured + caveats) without claiming primary-ranking-grade reliability.
≈ = extrapolated AA Eval Cost for an older same-brand snapshot not in our 23-model AA pool. Examples: GPT-5.3-Codex (≈$1,995, extrapolated from GPT-5.4's $2,851 with a Codex variant adjustment), Claude Sonnet 4.5 (≈$3,565, extrapolated from Sonnet 4.6's $3,959 with a generation-decay adjustment), Claude Opus 4.5 (≈$4,470, similarly), GPT-5.2 (≈$2,390). The model-tier ordering is preserved with high confidence; the absolute value is approximate within a brand-line. Plain $ values (no ≈) are exact AA Eval Cost figures from the current 23-model pool.
Three structural cases:

1. Multi-model entries. Junie CLI uses four different underlying models within the same submission (Gemini 3 Flash + Gemini 3.1 Pro + Claude Opus 4.6 + GPT-5.3-Codex). No single AA Eval Cost figure represents the blended cost.

2. Subscription pricing. Amazon Q Developer charges $19/month flat regardless of task volume. There's no per-task token decomposition published.

3. Compute-unit pricing. Devin 2.0 bills in Cognition AI's Agent Compute Units (1 ACU ≈ 15 minutes of active work, ~$8–9/hour). Translation to token cost requires assumptions that no public source supplies.

Value = score points per $1,000 of AA Eval Cost. So an 80.2% agent on Gemini 3.1 Pro ($892 AA) scores 80.2 ÷ 0.892 = 89.9 score-points-per-$K. Tier thresholds:

Outstanding ≥ 60 · Excellent ≥ 30 · Good ≥ 15 · Fair ≥ 5 · Premium < 5

Because Cost is uniform AA Eval Cost across all rows, the Value column compares apples-to-apples within the table. A row's Value tier reflects its actual relative cost-efficiency, not the choice of which methodology was applied to it.
The slider and column sort are mutually exclusive. At slider = 0% the table reflects raw benchmark score (default). Moving the slider toward 50% blends a normalized cost metric into the ranking via composite = (1 − w) × score + w × costNorm, where costNorm linearly maps the cheapest cost in the table to 100 and the most expensive to 0. Clicking any column header resets the slider to 0% and applies your column sort. Moving the slider clears the column-sort arrow and re-ranks by composite. The active control is always identifiable: if a header has a colored ↑ or ↓ arrow, column sort is active; if all arrows are gray ↕, the slider is in charge.
TB2.0 and SWE-bench Verified test fundamentally different capabilities and attract entirely different agent populations. TB2.0 is multi-step terminal/CLI workflows in Docker environments — system administration, data processing, ML pipeline, security tasks. SWE-V is GitHub issue resolution on Python repositories — code-repair agents fixing real bugs in mature codebases. Only one entry (Factory Droid + GPT-5.3-Codex) appears in both leaderboards' top tier. Forcing them into a single composite would average across structurally incomparable workloads and produce a misleading "general coding agent" ranking that doesn't predict performance on either real-world task type. We keep them visually parallel but mathematically separate, and surface the cross-benchmark Pareto frontier as the most honest combined view the data supports.
